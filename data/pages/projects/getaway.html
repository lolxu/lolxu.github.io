    • Getaway

    This is a mobile game developed from a class project for ITP 382 - Mobile Game Programming at USC. This project innovates on the touch control of mobile devices and use it in multiple ways as the main form of control.

    The getaway car will passively drive to the checkpoints, but it will need the players' direct input to evade cops by drifting. There will also be random powerups that can be obtained from getting to special checkpoints. Get to enough checkpoints to progress to next level.

    The project uses Unity NavMesh Agent to implement the pathfinding for the getaway vehicle, and the game detects touch controls to create forces that can push the getaway vehicle. To prevent race condition between the physics and NavMesh agent, I try to enable and disable the agent based on the amount of force the getaway vehicle is experiencing. I made sure the physics will not interfere with the NavMesh agent's pathfinding. I also coded a custom Render Feature in Unity to use a custom HLSL shader for the pixelated effects.
