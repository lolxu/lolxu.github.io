    • DirectX Game Engine

    This is a DirectX11 Game Engine that received continued development from my ITP485 - Game Engine Programming class.

    Implementing this game engine allowed me to learn the basics of rendering in the context of a graphics library (D3D11), and how basic functionalities like lighting, animations, and physics collisions can be implemented in a real game engine.

    Here are some features that I have implemented so far:

    • Rendering with ID3D11 buffers (Vertex Buffer, Index Buffer, etc...)
    • Skeletal Animations with Animation Blending
    • JSON format asset import (including Mesh, Animation, Level, etc...)
    • Multiple Render Targets for Bloom post-processing effect
    • HLSL shader support (thanks to the D3D11 library)
    • Collision Detection such as Raycast vs AABB and AABB vs AABB

    Notes on Rendering

    The most important thing I have learned from my course and the development of this engine is how rendering is achieved in modern game engines. We are using different buffers to store important information like vertices, lighting, z-depth, etc... and sending those buffers to the GPU for it to render. Thanks to the powerful D3D11 rendering pipeline, a lot of the foundational features like rasterization and GPU communications are taken care of. We only need to send over the correct buffers.

    Another crucial thing I have learned is about shader supports. Shaders are really powerful things! For this engine, a lot of the rendering features are made through HLSL shaders. I made a Phong Lighting HLSL shader that implements the Phong lighting model, Mesh shader that puts the mesh in world space, and even a Skinned shader that can skin a character model with the provided skeleton data. All shaders are made up of a Vertex Shader (for modifying vertices) and a Pixel Shader (for modifying color and per pixel data).
